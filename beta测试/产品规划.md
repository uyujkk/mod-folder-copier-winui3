# Beta 产品规划

## 总目标
把当前应用逐步升级为：
- 一级侧边栏负责模块导航
- 二级侧边栏负责仓库切换和新增仓库
- 仪表板显示全部仓库的预览数据
- 仓库页复用现有单仓库管理逻辑
- 在线页逐步接入站点浏览、安装和更新追踪

## 设计原则
- 不直接改动正式版目录
- 所有新想法先在 `beta测试/` 中验证
- 先完成本地多仓库架构，再接在线能力
- 每个仓库的核心交互逻辑保持一致，只是路径和来源配置不同

## 仓库模型
`WorkspaceRepository`

字段：
- `Id`
- `Name`
- `SourcePath`
- `TargetPath`
- `LauncherPath`
- `OnlineSourceSite`
- `OnlineCategoryId`
- `Notes`

说明：
- `Name` 由用户自定义
- 在线相关字段在一期先完成落库和编辑
- 二期开始真正用于在线页面和数据抓取

## 导航结构
### 一级侧边栏
- 仪表板
- 仓库
- 在线 Mod
- 更新
- 设置

### 二级侧边栏
- 仪表板下显示 `全部仓库` 和仓库列表
- 仓库下显示仓库列表
- 在线 Mod 下显示仓库列表
- 设置下保留当前仓库上下文，但主区展示全局设置

## 一期工程
### 目标
- 完成多仓库 Beta 外壳
- 完成一级 / 二级侧边栏
- 完成仪表板、仓库页、设置页
- 补齐仓库增删改切换和配置落盘

### 一期完成情况
- 已完成一级侧边栏：仪表板 / 仓库 / 设置
- 已完成二级侧边栏和仓库操作
- 已完成仪表板摘要卡片
- 已完成仓库页复用现有单仓库逻辑
- 已完成仓库详情编辑
- 已完成 GitHub 更新检查和更新内容显示

### 一期剩余
- 只剩体验层打磨和代码清理
- 不再有结构性缺口

## 二期工程
### 目标
- 接入在线 Mod 页面
- 将在线来源和当前仓库做正式绑定
- 搭好后续列表抓取、详情预览、安装流程的基础结构

### 二期拆分
1. 在线页骨架
- 页面结构
- 仓库来源映射展示
- 准备状态展示

2. 数据准备
- 站点来源字段
- 分类 ID 字段
- 备注说明
- 本地记录远程来源的预留结构

3. 列表能力
- 读取当前仓库来源配置
- 拉取站点列表
- 支持搜索、排序、详情预览

4. 安装闭环
- 下载压缩包
- 导入当前仓库
- 记录来源链接和远程 ID

## 三期工程
- Mod 更新检测
- 仓库级更新状态
- 已安装 Mod 与远程来源对比
- 打开详情页或下载新版

## 四期工程
- 批量操作
- 仓库排序 / 固定
- 缓存管理
- 仓库导入导出
- 更完整的错误恢复和产品化体验
