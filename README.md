# Mod 文件复制器 WinUI 3 版 v2.2.3

> Non-official notice:  
> This project is an unofficial fan-made tool. It is not affiliated with, endorsed by, or sponsored by XXMI, any game publisher, or any related developers.

这是一个运行在 Windows 上的 WinUI 3 图形化工具，用于管理两层 Mod 文件夹、复制或移除第二层 Mod、导入 ZIP、记录每个 Mod 的快捷键与描述、设置预览图，以及快速启动外部启动器。

当前版本：

- 显示版本：`v2.2.3`
- 文件版本：`2.2.3.0`

## 1. 主要功能

- 展示 Mod 存储文件夹中的第一层和第二层文件夹
- 选中第二层 Mod 后，在右侧显示默认预览图
- 双击第二层 Mod，或点击按钮，将该 Mod 整体复制到目标文件夹
- 如果目标文件夹中已经存在同名文件夹，再次操作会移除该 Mod
- 支持将 `.zip` 文件直接导入到当前选中的第二层 Mod 文件夹
- 支持为每个第二层 Mod 单独记录快捷键和描述
- 支持拖动图片到预览区，自动设置为当前 Mod 的预览图
- 支持为每个 Mod 绑定独立链接，并通过默认浏览器快速访问
- 支持配置并运行外部启动器，例如 `XXMI Launcher`
- 支持浅色 / 深色主题切换
- 支持中文 / English 界面切换

## 2. 适用场景

适合以下情况：

- 你有一个总 Mod 目录，里面按分类存放了多层文件夹
- 你只想把某一个具体 Mod 临时复制到游戏或启动器读取目录
- 你希望在切换 Mod 时顺便查看预览图
- 你需要给不同 Mod 记录对应的按键和说明
- 你希望从压缩包直接把文件解压到某个 Mod 文件夹里

## 3. 目录结构要求

程序按“两层目录”工作。推荐结构如下：

```text
Mod 存储文件夹
├─ 分类A
│  ├─ Mod1
│  ├─ Mod2
│  └─ Mod3
├─ 分类B
│  ├─ Mod4
│  └─ Mod5
└─ 分类C
   └─ Mod6
```

其中：

- 第一层文件夹：分类目录
- 第二层文件夹：实际操作的 Mod 文件夹

程序真正复制、移除、导入和绑定链接的对象是第二层文件夹。

## 4. 运行方式

正式运行入口：

- `dist/ModFolderCopier.exe`

WinUI 主程序：

- `dist/WinUI3/ModFolderCopier.WinUI.exe`

通常直接双击 `ModFolderCopier.exe` 即可。

## 5. 界面说明

### 5.1 顶部路径模块

包含三项路径：

- Mod 存储文件夹
- 目标文件夹
- 启动器地址

说明：

- Mod 存储文件夹：你的总 Mod 目录
- 目标文件夹：实际复制目标目录，通常是启动器或游戏读取的 Mod 目录
- 启动器地址：外部启动器程序，例如 `XXMI Launcher.exe`

### 5.2 操作按钮区

包括：

- 刷新目录
- 导入 ZIP 到当前第二层
- 运行启动器
- 复制当前第二层文件夹

### 5.3 状态卡片区

显示：

- 第一层目录数
- 第二层目录数
- 当前复制状态
- 当前第二层文件夹

### 5.4 第一层文件夹列表

用于选择分类目录。

### 5.5 第二层文件夹列表

用于选择具体 Mod。双击这里的项目，会执行复制或移除操作。

### 5.6 快捷键与描述区

这里不是全局热键注册器，而是当前 Mod 的快捷键说明区。你可以为当前选中的 Mod 填写：

- 快捷键
- 描述

说明：

- 支持单个按键，例如 `1`、`Q`、`F1`
- 也支持组合键，例如 `Ctrl+1`
- 默认显示 1 行，最多支持 10 行

### 5.7 预览图与链接区

预览区用于显示当前第二层 Mod 的图片预览。程序优先查找以下命名的图片：

- `preview.*`
- `cover.*`
- `thumbnail.*`
- `image.*`

如果这些都没有，会尝试显示当前 Mod 文件夹里的第一张图片。

预览图下方还提供 Mod 链接模块：

- 每个 Mod 都可以绑定独立链接
- 点击快速访问按钮时，会调用系统默认浏览器打开该链接

## 6. 详细使用步骤

### 6.1 第一次打开程序

1. 运行 `ModFolderCopier.exe`
2. 设置 Mod 存储文件夹
3. 设置目标文件夹
4. 如果需要，设置启动器地址
5. 点击“刷新目录”

### 6.2 选择并复制一个 Mod

1. 在“第一层文件夹”中点击一个分类
2. 在“第二层文件夹”中选中一个具体 Mod
3. 查看右侧预览图，确认目标 Mod
4. 双击该 Mod，或点击“复制当前第二层文件夹”

执行结果：

- 如果目标文件夹里没有同名目录：程序会复制该 Mod
- 如果目标文件夹里已经有同名目录：程序会移除该 Mod

### 6.3 导入 ZIP 到当前 Mod

1. 先选中一个第二层 Mod
2. 点击“导入 ZIP 到当前第二层”
3. 选择一个 `.zip` 文件

程序会把压缩包中的内容解压到当前 Mod 文件夹中。

### 6.4 设置预览图

有两种方式：

方式一：手动放入图片

把图片放进对应 Mod 文件夹，并命名为以下任意一种：

- `preview.png`
- `cover.jpg`
- `thumbnail.webp`
- `image.png`

方式二：直接拖动图片

1. 先选中一个第二层 Mod
2. 从资源管理器中拖动图片到右侧预览区
3. 程序会自动把它复制到当前 Mod 文件夹中，并设置为新的预览图

### 6.5 配置快捷键与描述

1. 选中一个第二层 Mod
2. 在“快捷键与描述”区域中填写内容

每一行包含：

- 快捷键，例如 `Ctrl+1`
- 描述，例如“切换到常用版本”

如需更多行，可点击“新增一行”。

### 6.6 配置 Mod 链接

1. 选中一个第二层 Mod
2. 在预览图下方的链接模块中输入对应链接
3. 点击快速访问按钮，可直接打开该链接

### 6.7 运行外部启动器

1. 在顶部“启动器地址”中选择程序
2. 点击“运行启动器”

程序会直接启动该 EXE，并使用它所在目录作为工作目录。

## 7. 配置文件说明

程序运行时会在运行目录生成配置文件，例如：

- `dist/WinUI3/config.ini`

配置中会保存：

- Mod 存储文件夹路径
- 目标文件夹路径
- 启动器路径
- 当前主题
- 当前语言
- 每个 Mod 对应的快捷键与描述
- 每个 Mod 对应的链接

## 8. 如何重新构建

源码入口：

- `WinUI3/`
- `WinUILauncher.cs`
- `build_winui.bat`

直接构建：

```powershell
cmd /c build_winui.bat
```
